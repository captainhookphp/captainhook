Composer integration

If you want to make sure everybody in your team installs the git hooks to their local repository you have to options.

1. Composer plugin

Instead of requiring captainhook/captainhook in your composer.json file you can require captainhook/plugin-composer. This will install CaptainHook and a composer plugin, that makes sure the hooks get activated for everyone's local repository. 

{
  "require-dev": {
    "captainhook/plugin-composer": "^4.0"
  }
}

2. Post install command

You can use the post-autoload-dump command just add the following lines to your projects composer.json file. 

{
  "scripts": {
    "post-autoload-dump": "\\CaptainHook\\App\\Composer\\Cmd::setup"
  }
}

Including this lines, Composer will execute the CaptainHook install command every time composer install or update is executed.

A word of warning

It is still possible to commit without invoking the hooks. So make sure you run appropriate backend-sanity checks on your code!
